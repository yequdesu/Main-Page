Act 可见性：条件渲染 vs visible 切换

R3F 中如何表达"Act 1 结束，Act 2 开始"

A

条件渲染（React 挂载/卸载）

Act 变化时销毁旧组件、挂载新组件。React 语义最干净，天然对应 build/exit 生命周期。

{sp < 0.45 && <Act1OceanVoyage sp={sp} />}
{sp >= 0.40 && sp < 0.85 && <Act2GridTransition sp={sp} />}
{sp >= 0.85 && <Act3ContentPhase sp={sp} />}
B

visible prop 切换（始终挂载）

所有 Act 组件始终存在于场景中，通过 visible prop 控制可见。不重新创建几何体和材质。

<Act1OceanVoyage visible={needsAct1(sp)} sp={sp} />
<Act2GridTransition visible={needsAct2(sp)} sp={sp} />
<Act3ContentPhase visible={sp >= 0.85} sp={sp} />
C

混合（交叉区双挂载 + visible 控制）

利用 Act 1→2（0.40–0.45）和 Act 2→3（0.85 附近）有交叉区，在交叉区内双挂载实现交叉淡入淡出。

R3F 社区对此的立场

R3F Performance Pitfalls：不要动态挂载/卸载
"Don't mount/unmount dynamically... recompiles materials, geometry, shaders every time. Use visibility instead."
— R3F 官方文档 · Performance Pitfalls

但该规则主要针对高频切换场景。Act 切换是低频操作（用户需要滚动相当距离才能触发），交叉区长度足够让 React 完成渲染。

关键数据：切换频率

切换 sp 阈值 切换性质 交叉区长度
Act 1 → Act 2 0.40–0.45 交叉淡化 0.05 sp ≈ 810px 滚动
Act 2 → Act 3 0.85 硬切换（有交叉区） ~0.01 sp ≈ 162px 滚动
