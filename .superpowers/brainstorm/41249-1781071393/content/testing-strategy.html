测试策略：测到什么程度？

R3F 项目测试的三层模型

三层测试模型

层级 测试内容 工具 成本 价值
L1 · 纯函数 位置/颜色/透明度计算、阈值判定、smoothstep、切线几何 vitest 极低 极高
L2 · R3F 组件 场景图结构、useFrame 逐帧行为、ref 绑定、事件触发 @react-three/test-renderer 中 中
L3 · 视觉回归 渲染输出截图对比、像素级验证 Puppeteer + headless-gl 高 低

两个候选方案

A

L1 全覆盖 + L2 关键路径

纯函数（behaviors/）100% 覆盖；R3F 组件仅对 Act 切换、点击聚焦等关键交互做测试。视觉回归测试不做。

投入产出比最高。behaviors 包含了所有容易出错的数学逻辑。

B

L1 + L2 全覆盖 + L3 关键帧

所有 R3F 组件测试场景图结构；对 7 个关键 sp 阈值截帧做视觉回归。CI 中跑完整测试套件。

最完整，但 L3 维护成本高——任何视觉微调都会导致截图失效。

L1 测试示例（纯函数，零依赖）

// behaviors/__tests__/positionUpdate.test.ts
import { updateParticlePosition } from '../positionUpdate'

describe('updateParticlePosition', () => {
  it('returns Act 1 float position at sp=0', () => {
    const p = { wx: 1, wy: 2, wz: 3, ph: 0, orbitAngle: 0, orbitR: 3.6 }
    const { x, y } = updateParticlePosition(p, 0, 0)
    expect(x).toBeCloseTo(1.25)
  })

  it('fully transitions to orbit at sp=1', () => {
    const p = { wx: 5, wy: 0, wz: 5, ph: 0, orbitAngle: 0, orbitR: 3.6 }
    const { x, z } = updateParticlePosition(p, 0, 1)
    expect(x).toBeCloseTo(3.6)   // cos(0) * 3.6
    expect(z).toBeCloseTo(-16)   // SCENE_CENTER_Z
  })
})
