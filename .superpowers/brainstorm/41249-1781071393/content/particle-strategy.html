粒子系统：InstancedMesh vs 独立 Mesh vs 混合

135 个粒子中的 3 个主行星 + 132 个碎片

当前实现与 R3F 选项对比

InstancedMesh · 全部 独立 Mesh · 全部 混合（推荐）
Draw Calls 1（最优） 135 3 主行星 + 1 InstancedMesh = 4
逐粒子透明度 需自定义 shader material.opacity 碎片用 shader，主行星用 material
逐粒子颜色 instanceColor（r139+） material.color 两者皆可
点击检测 需额外 Raycaster R3F onClick 天然支持 主行星用 onClick，碎片用 NDC
标签跟随 需手动同步矩阵 读 mesh.position 读主行星 ref.position
遮挡淡化逻辑 需在 shader 中实现 CPU 侧，简单 碎片在 shader 中，CPU 辅助

推荐方案：混合

3 主行星 = R3F Mesh 组件 + 132 碎片 = InstancedMesh
// actors/DustField.tsx
<group ref={dustGroup}>
  {/* 3 主行星：独立 Mesh，完整 React 生命周期 */}
  {planets.map(p => (
    <Planet key={p.index} data={p} onClick={handleClick} />
  ))}

  {/* 132 碎片：InstancedMesh，单 Draw Call */}
  <instancedMesh ref={debrisRef} args={[null, null, 132]}>
    <sphereGeometry args={[0.015, 10, 8]} />
    <meshBasicMaterial />
  </instancedMesh>
</group>

// hooks/useDustUpdate.ts — 一帧更新所有粒子
useFrame((_, delta) => {
  // 主行星：直接操作 ref
  planets.forEach(p => p.ref.position.copy(orbitPos))

  // 碎片：批量写入 InstancedMesh 矩阵 + instanceColor
  const m = new Matrix4()
  debris.forEach((d, i) => {
    m.compose(d.position, d.quaternion, d.scale)
    debrisRef.current.setMatrixAt(i, m)
  })
  debrisRef.current.instanceMatrix.needsUpdate = true
})

社区参考

适用 InstancedMesh

  • R3F 官方示例：1000+ 草叶 / 星场粒子
  • Galaxy Voyager：星系粒子用 InstancedMesh + custom shader 实现逐星颜色/亮度
  • 门槛：~500+ 实例时 Draw Call 差异才显著

适用独立 Mesh

  • Drei 的 Sparkles 组件：每个粒子独立 Mesh（通常 < 200）
  • 大多数 R3F 作品集网站：行星/装饰物用独立 Mesh，代码简单
  • 3 个主行星需要 onClick、ref、标签——独立 Mesh 天然支持
