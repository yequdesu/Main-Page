状态管理架构：三层状态如何映射到 R3F

当前 Vanilla 的三层状态 → R3F + Zustand 的对应关系

当前状态分布（Vanilla Three.js + Vue）

层级 存放位置 更新频率 示例
UI 状态 Vue ref / computed 低频（交互触发） isAct3Focused, overlayData, brandTextVisible
渲染状态 模块级变量 逐帧（60fps） _hoveredIdx, _focusedPlanetIdx, _labelOpacityCurrent, 预分配对象池
动画引擎 GSAP tween 对象 实时（tween 驱动） _focusTween, _clickTween, _physVelocity

三个候选方案

A

Zustand 全状态

所有三层状态都进 Zustand store，UI 通过 hook 订阅，渲染状态通过 getState() 在 useFrame 中瞬态读取，GSAP 直接更新 store。

R3F 社区最广泛采用（Galaxy Voyager 220+ 星系均用此模式）

B

Zustand（渲染）+ React（UI）分层

Zustand 仅管理渲染状态（scrollProgress, focusedIdx, hoveredIdx）；UI 状态用 React useState/context；GSAP 通过 ref 直接操作 DOM。

接近 R3F 官方文档推荐的"五类出口"模式

C

React Context + useRef 极简

不用 Zustand，渲染状态用 module-level refs（类似当前做法），UI 状态用 React context。零额外依赖，但缺少瞬态读取 API。

最小依赖，适合状态简单的项目

方案 B 在实际代码中的样子
