2D 叠加层的库选择

GSAP 驱动动画 + 选择合适的渲染层

GSAP 怎么用在这个场景？

GSAP 不只是 DOM 动画库——它可以 tween 任意 JS 对象属性。用 GSAP Timeline + ScrollTrigger 来驱动 Canvas 2D 的绘制参数（线位置、透明度、旋转角、阵列相位），效果完全可控。

🎯

GSAP + Canvas 2D

纯手绘，完全控制

✓ 零新依赖（GSAP 已在项目中）
✓ GSAP tween 控制所有绘制参数
✓ 完全自由——任何几何形、任何曲线
△ 需要手写所有绘制逻辑
△ 没有场景图，需要自己管理元素状态
🎨

p5.js

创意编程标准库

✓ 专为几何/阵列/生成艺术设计
✓ push()/pop() 变换栈—天然适合阵列旋转
✓ map()/lerp() 等工具函数
△ 新增依赖 (~800KB)
✗ 自带 draw 循环，需与项目 rAF 协调
📐

GSAP + SVG

声明式矢量，DOM可检视

✓ GSAP 直接操作 SVG 元素属性
✓ 元素可检视、可调试
✓ 天然响应式缩放
△ 阵列元素多时 DOM 节点开销大
△ 复杂曲线排布需要 <animateMotion> 或 JS 计算
🖼️

Two.js

轻量 2D 场景图

✓ 有场景图——不需要手动管理状态
✓ 支持旋转/缩放/平移变换
✓ 轻量 (~100KB)，Canvas/SVG/WebGL 多后端
△ 新增依赖
△ GSAP 需通过 tween 对象间接控制 Two.js 属性

我的建议：GSAP + Canvas 2D

零新依赖，GSAP Timeline 把 scrollProgress 映射到 Canvas 绘制参数。阵列的旋转/位移直接用 GSAP tween 驱动 JS 对象，Canvas 每帧读取这些对象的值来绘制。这个模式在创意编程中非常常见，完全够用。
