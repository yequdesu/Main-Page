技术实现方案对比

确认需求：测地线框 + 射线+微粒混合 + 边缘色散。两种实现路径可选。

方案 1

纯 ShaderMaterial 方案 (推荐)

所有效果都在恒星自身的材质着色器中完成，不引入任何后处理管线。

架构

  • 测地线框球体 — 自定义 ShaderMaterial，fragment shader 中基于视线角度计算红-蓝色散偏移
  • 射线 (40条) — BufferGeometry LineSegments，ShaderMaterial 按长度渐变透明度 + 末端微红
  • 微粒 (~100个) — THREE.Points + 自定义 ShaderMaterial，离球体越远越暗

优点

  • 零额外渲染 pass，性能最优
  • 不需要 EffectComposer / 后处理
  • 色散效果精确控制在恒星上，不影响轨道环和星球
  • 实现简单，改动集中在 ~150 行内
  • 与现有 Act 系统完美契合

缺点

  • 色散仅限恒星本身，无全屏"热浪扭曲"
  • 无法做跨物体的屏幕空间效果
方案 2

ShaderMaterial + 局部热浪扭曲层

方案 1 的全部内容 + 一个额外的全屏透明 Quad，在恒星屏幕位置产生径向热浪扭曲。

架构

  • 方案 1 的所有组件保持不变
  • 新增：HeatDistortQuad — 全屏透明平面，渲染顺序在场景最后
  • Quad 的 fragment shader 采样场景深度/颜色，只在恒星投影位置周围做径向 UV 偏移 + 微红移
  • 需要将恒星 3D 位置投影到屏幕 UV 坐标传入 uniform

优点

  • 真正的"高温扭曲空间"效果，恒星周围画面轻微变形
  • 视觉上更接近真实恒星的热辐射感
  • 不需要完整的 EffectComposer 管线

缺点

  • 额外的渲染 pass，性能开销约 +15-20%
  • 扭曲影响轨道环和星球——可能干扰可读性
  • 实现更复杂（屏幕投影、深度采样）
  • 边缘情况处理（恒星在屏幕外时的 fallback）
