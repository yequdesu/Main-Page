几何恒星 — 最终设计方案

测地线框 + 射线 + 微粒 | 暂不实现色散 | 方案1纯 ShaderMaterial (标准材质)

架构总览

_starGroup 组件树 (替换当前色球+光晕)
_starGroup ← position (0, -1.0, -16) / renderOrder=0
├── geodesicWire ← LineSegments / 二十面体线框
│ Geometry: IcosahedronGeometry(0.55, 4)
│ Material: LineBasicMaterial(#e8c888, opacity→1.0)
├── rayGroup ← Group / ~40条射线
│ 每条: BufferGeometry(2点) Line, 长度 0.3~1.5
│ 起点 in 球面, 终点沿法线延伸
│ Material: LineBasicMaterial, opacity 按长度渐变
└── particlePoints ← THREE.Points / ~100微粒
Geometry: BufferGeometry, 球壳内随机分布
Material: PointsMaterial, 小圆点, AdditiveBlending

关键参数

测地线框IcosahedronGeometry(0.55, 4) → EdgeGeometry → LineSegments
线框颜色#e8c888 (暖金) → Act 3 fade in 至 opacity 0.85
射线数量~40条，球面均匀分布 (斐波那契球)
射线长度0.3 ~ 1.5 单位，随机 + 时间微动
微粒数量~100个，分布在半径 0.55~1.2 球壳
微粒大小2~4px，AdditiveBlending 柔和发光
动画线框绕 Y 轴慢旋 (~0.15 rad/s)，射线长度呼吸，微粒漂浮
色散暂不实现，预留 fragment shader 接口

与现有代码的集成

替换 act3.build() 中 lines 1104-1158 的当前 _starGroup 创建逻辑。对外接口不变：

  • _starGroup.visible toggle 保持一致
  • 透明度通过各子组件的 material.opacity 控制
  • act3.dispose() 中遍历清理所有子对象
  • 不引入新的依赖或渲染 pass
