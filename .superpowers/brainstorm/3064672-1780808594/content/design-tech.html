如何在代码中落地这个叠加层

技术方案：第二 Canvas (2D Context)

<canvas> 叠加层 · 2D Context · z-index: 1
<canvas> Three.js · z-index: 0

为什么不用 SVG？

  • 阵列动画需要大量元素，SVG DOM 节点太重
  • Canvas 2D 的 globalCompositeOperation 天然支持混合模式
  • 与 Three.js 同样的 requestAnimationFrame 循环

为什么不用 CSS？

  • 复杂的几何阵列（沿曲线排列 + 相位旋转）CSS 无法表达
  • 构造线的精确像素级绘制需要 Canvas

组件结构

src/
├── App.vue ← 传递 scrollProgress 给叠加层
├── components/
│ ├── LighthouseScene.vue ← 现有 3D 场景（不动）
│ ├── CompositionOverlay.vue ← ★ 新增：平面构成叠加层
│ └── AppFooter.vue
└── composition/ ← ★ 新增目录
├── constructionLines.js 构造线：对角/三分/模数/准星/角度标记
├── geometricArrays.js 几何阵列：长方体阵列/点阵/线组/角标
├── postEffects.js 后效果：反色/混合模式/遮罩
└── actBehavior.js 各 Act 的出场/动画/离场逻辑

数据流

App.vue 已有 scrollProgress (0–1)，直接作为 prop 传入 CompositionOverlay.vue，无需新增状态管理。

scrollProgress: 0.00 ──── Act 1 ──── 0.45 ──── Act 2 ──── 0.85 ──── Act 3 ──── 1.00
构造线: 淡入 → 渐显 → 完整
几何阵列: 无 → 长方体阵列沿水平 → 长方体沿轨道曲线 + 径向线组
